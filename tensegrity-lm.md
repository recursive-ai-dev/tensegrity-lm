The file is at `/mnt/user-data/outputs/tensegrity_lm.py`. Let me present it.## TensegrityLM — Algorithmic Designation

**TensegrityLM**: named for the structural-engineering principle where a system holds shape through balanced tension and compression rather than rigid load-bearing members. Every layer carries a learnable "gain" that has to balance against a homeostatic regularizer pulling activation energy toward a set-point — the network is held together by opposing forces, not fixed weights.

### Mapping Rationale

| Domain concept | Irreducible primitive | Role in TensegrityLM |
|---|---|---|
| Biology — homeostasis | Activation-RMS set-point loss per sublayer | `homeo_loss`: penalizes deviation of attention/FFN output RMS from target 1.0 |
| Geometry — rotation | RoPE (rotary position embedding) | Position encoding via 2D rotation of query/key pairs |
| Space — locality vs. global reach | Alternating sliding-window / full causal masks | Layers 1,2 local (window=128), every 3rd layer global |
| Evolution — depth-dependent mortality | Linear-schedule stochastic layer drop | Drop probability scales 0→`layerdrop_max` with depth |
| Social Dynamics — specialist routing | Top-k MoE with Switch load-balancing loss | Every 2nd layer routes tokens to 2-of-4 SwiGLU experts |
| Psychology — metacognition | Confidence head (sigmoid, predicts P(correct)) | Auxiliary BCE loss; usable for adaptive sampling temperature |
| Algorithms — grouped-query attention | GQA (6 query heads / 2 KV heads) | Reduces KV cache while preserving most quality |
| Algebraic Math — normalized residual scaling | RMSNorm + learnable per-sublayer gain (DeepNet-style init) | Pre-norm blocks with `1/√(2L)`-initialized residual gains |
| Physics — annealing | Cosine LR schedule with warmup | Standard but framed as "temperature annealing" of the optimizer |

### Architectural Logic (~420 words)

TensegrityLM is a byte-level (vocab=259: 256 bytes + BOS/EOS/PAD), decoder-only transformer that requires zero tokenizer training — any UTF-8 or binary file is immediately ingestible. Each of the `n_layer` blocks is pre-norm (RMSNorm) and contains two sublayers: causal self-attention and a feed-forward network, each wrapped in a residual connection scaled by a **learnable gain** initialized to `1/√(2·n_layer)`.

The novel coupling is between three independently-established mechanisms that are rarely combined this way. First, **attention pattern alternation**: layers cycle between a local sliding-window mask (width 128) and a full causal mask every `global_every` layers, giving cheap local mixing most of the time with periodic global context refresh — a Longformer/Mistral-style pattern, but here it's load-bearing for the homeostasis term below, since local layers produce systematically different activation statistics than global ones.

Second, **homeostatic residual regularization**. For every sublayer output, the model computes `activation_rms(out)` and penalizes `(rms - target)²`. This isn't normalization (which is already handled by RMSNorm at the input) — it's a *target* on the output magnitude *before* it gets scaled by the learnable gain and added to the residual stream. The gain and the homeostasis loss are in tension: the gain wants to grow to let the sublayer contribute more, while homeostasis caps how "loud" that contribution gets relative to the residual stream's running scale. This is the "tensegrity" — no single component dictates the residual stream's energy budget; it emerges from the equilibrium between gains and the set-point penalty, layer by layer.

Third, **depth-scaled stochastic depth** interacts with this equilibrium: deeper layers (which have larger initialized gains relative to earlier layers under DeepNet-style init, and operate on a residual stream that's already accumulated more energy) are dropped more often during training (`layerdrop_max · layer_idx/(n_layer-1)`). This means the homeostasis target has to be satisfiable by *partial* layer subsets too — the network can't rely on every layer firing every step, which empirically tends to make residual-gain regularizers more robust (this is the "Evolution" framing: layers that survive stochastic ablation are the ones whose contribution generalizes).

The MoE layers (every 2nd) add a fourth tension: routed experts contribute to the same homeostasis-regulated residual stream, but their output statistics depend on which 2-of-4 experts a token gets routed to — so the load-balancing auxiliary loss and the homeostasis loss are jointly shaping both *which* experts fire and *how loud* they're allowed to be.

The confidence head is the cheapest addition: a linear projection from the final hidden state predicting whether the argmax prediction matches the target, trained with BCE. At inference it can rescale sampling temperature per-token — high confidence → lower temperature (more greedy), low confidence → higher temperature (more exploration).
